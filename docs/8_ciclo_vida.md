**CICLO DE VIDA DE ENGENHARIA DE USABILIDADE**

1. **Características da Plataforma**

| Característica | Descrição |
| :---- | :---- |
| Descrição do Software | **Frontend**: A interface do usuário será desenvolvida como uma aplicação web usando JavaScript e a biblioteca React JS. **Backend**: O serviço será construído em .NET 8.0 com C#, utilizando a biblioteca FaceAISharp para reconhecimento facial. **Infraestrutura**: O sistema será hospedado na nuvem Amazon Web Services (AWS), utilizando serviços como EC2 para computação, SQS para mensageria e RDS/MongoDB para banco de dados. **Ambiente de Usuário**: A aplicação será acessada via navegadores web padrão em sistemas operacionais corporativos.  |
| Descrição do Hardware | **Lado do Usuário**: Computadores de mesa padrão localizados nos centros de controle e escritórios administrativos do Metrô. Para os Operadores de Câmeras, espera-se o uso de estações de trabalho com múltiplos monitores de alta resolução. **Lado do Servidor**: A infraestrutura é baseada em instâncias virtuais AWS EC2, com poder computacional e armazenamento definidos conforme a demanda. **Dispositivos de Captura**: Câmeras de vigilância IP instaladas nas estações do Metrô. |
| LISTA DE Capacidades da Plataforma (com explicação) | **Alta Escalabilidade e Disponibilidade**: Por utilizar a infraestrutura da AWS, o sistema pode escalar seus recursos (processamento, armazenamento) para lidar com grandes volumes de dados de múltiplas câmeras, garantindo operação contínua 24/7, o que é crítico para o Metrô. **Interface Web Moderna e Acessível**: O uso de React JS permite a criação de uma interface rica, interativa e responsiva, que pode ser acessada de qualquer terminal com um navegador, facilitando a distribuição e manutenção do software. **Processamento Assíncrono de Dados**: Com o AWS SQS, o sistema pode processar as imagens e dados de forma distribuída e assíncrona, garantindo que a interface do usuário permaneça ágil e responsiva, mesmo sob alta carga de processamento no backend. |
| LISTA DE Restrições da Plataforma (com explicação) | **Dependência de Conectividade**: Sendo um sistema distribuído e baseado na nuvem, seu funcionamento depende totalmente de uma conexão de rede estável e de alta velocidade entre as estações (câmeras), os centros de controle (usuários) e os servidores da AWS. **Custo Operacional Variável**: A utilização de serviços em nuvem gera custos contínuos que variam conforme o uso. Picos de processamento de imagens ou armazenamento de dados podem levar a um aumento significativo nos custos operacionais para o Metrô. **Limitações do Reconhecimento Facial**: A precisão dos algoritmos da biblioteca FaceAISharp é dependente da qualidade das imagens das câmeras. Fatores como iluminação baixa, ângulos desfavoráveis e oclusão parcial do rosto são restrições que podem impactar a qualidade dos dados exibidos na interface. |

3. **Metas de Usabilidade**
   
    1. **Qualitativo**
      * Clareza da Informação: A interface deve traduzir dados brutos de fluxo em visualizações (gráficos, mapas de calor) que sejam intuitivas e permitam a rápida identificação de padrões e anomalias pelos analistas e administradores.
      * Responsividade em Tempo Real: As interações na tela de monitoramento do operador devem ser imediatas. Ações como selecionar uma câmera, dar zoom ou registrar uma ocorrência não devem apresentar atrasos perceptíveis.
      * Flexibilidade e Controle para o Usuário: A interface deve permitir que os analistas personalizem seus dashboards, salvem filtros complexos e exportem dados brutos, dando-lhes controle para realizar análises aprofundadas.
      * Prevenção de Erros Críticos: Funções administrativas, como cadastrar um novo usuário ou configurar uma câmera, devem ter um design que guie o usuário e exija confirmação para ações destrutivas, minimizando a chance de erros operacionais.
      * Consistência da Interface: Todos os módulos do sistema (dashboards, relatórios, configurações) devem seguir os mesmos padrões de design, terminologia e comportamento, reduzindo a carga cognitiva e acelerando a curva de aprendizado.
